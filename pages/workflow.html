Development Workflow & Code Review

Git workflow, code review process, and pre-release checklist.

Follow this branch structure: main → staging → feature/bugfix.

Branch Workflow 
# Create feature branch from staging
git checkout staging
git pull origin staging
git checkout -b feature/new-component

# Work on feature
git add .
git commit -m "feat: add new modal component"

# Push and create PR to staging
git push origin feature/new-component

# After review and merge to staging, test
# Then create PR from staging to main for release
