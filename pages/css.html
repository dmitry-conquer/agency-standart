CSS Standards & BEM Methodology

BEM naming convention, responsive design, and CSS best practices.

Block Element Modifier (BEM) ensures consistent, maintainable CSS.

BEM Structure 
/* Block */
.button { }

/* Element (block__element) */
.button__icon { }
.button__text { }
.button__loader { }

/* Modifier (block--modifier or block__element--modifier) */
.button--primary { }
.button--large { }
.button--disabled { }
.button__icon--spinning { }

/* HTML Example */
<button class="button button--primary button--large">
  <span class="button__icon button__icon--spinning"></span>
  <span class="button__text">Submit</span>
</button>

Use rem/em for typography and spacing. Use unified breakpoint system.

Example: Responsive Typography 
.card {
  padding: 1.5rem; /* 24px */
  margin-bottom: 2rem; /* 32px */
  font-size: 1rem; /* 16px */
  line-height: 1.6;
}

.card__title {
  font-size: 1.5rem; /* 24px */
  margin-bottom: 1rem; /* 16px */
  
  @include medium {
    font-size: 1.25rem; /* 20px */
  }
  
  @include small {
    font-size: 1.125rem; /* 18px */
  }
}

/* Use rem for spacing, em for relative sizing */
.button {
  padding: 0.75em 1.5em; /* Relative to font-size */
  font-size: 1rem; /* Base size */
}

Organize CSS following ITCSS methodology: Settings → Tools → Generic → Elements → Objects → Components.

File Structure 
styles/
├── settings/          # Variables, config
│   ├── _colors.scss
│   └── _typography.scss
├── tools/             # Mixins, functions
│   ├── _mixins.scss
│   └── _functions.scss
├── generic/           # Reset, normalize
│   └── _reset.scss
├── elements/          # Base HTML elements
│   └── _headings.scss
├── objects/           # Layout patterns
│   └── _container.scss
├── components/        # UI components
│   ├── _button.scss
│   ├── _card.scss
│   └── _modal.scss
└── utilities/         # Utility classes
    └── _spacing.scss
  • ✓ Never use inline styles (except dynamic values)
  • ✓ Avoid !important (use specificity instead)
  • ✓ Group related properties together
  • ✓ Use CSS custom properties for theming
  • ✓ Minimize nesting (max 3 levels)
Example: Well-Organized CSS 
/* Good: Organized, readable */
.card {
  /* Layout */
  display: flex;
  flex-direction: column;
  
  /* Spacing */
  padding: 1.5rem;
  margin-bottom: 2rem;
  
  /* Visual */
  background: var(--bg-secondary);
  border: 1px solid var(--border);
  border-radius: 0.5rem;
  
  /* Typography */
  font-size: 1rem;
  line-height: 1.6;
  
  /* Transitions */
  transition: all 0.2s ease;
  
  &:hover {
    border-color: var(--border-light);
    box-shadow: var(--shadow-md);
  }
}

/* Bad: Disorganized, hard to read */
.card { display: flex; padding: 1.5rem; background: #fff; border: 1px solid #ccc; }
.card:hover { border-color: #999; }
