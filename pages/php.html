PHP Standards & Security

PSR standards, security best practices, and code organization.

Always sanitize input and escape output. Never trust user data.

Example: Sanitization & Escaping 
// Sanitize input
$email = sanitize_email($_POST['email']);
$name = sanitize_text_field($_POST['name']);
$url = esc_url_raw($_POST['website']);
$textarea = sanitize_textarea_field($_POST['message']);

// Escape output
echo esc_html($user_input);
echo esc_attr($attribute_value);
echo esc_url($link_url);
echo esc_js($javascript_value);
echo wp_kses_post($html_content); // Allow safe HTML

// Database queries
global $wpdb;
$user_id = absint($_GET['user_id']); // Ensure integer
$query = $wpdb->prepare(
  "SELECT * FROM {$wpdb->prefix}users WHERE ID = %d",
  $user_id
);
$results = $wpdb->get_results($query);

// Never do this:
// echo $_POST['email']; // XSS vulnerability
// $wpdb->query("SELECT * FROM users WHERE ID = " . $_GET['id']); // SQL injection

Follow PSR-12 coding standards. Use namespaces and autoloading.

Example: Class Structure 
<?php
/**
 * Plugin Name: Example Plugin
 * Description: Example plugin following standards
 * Version: 1.0.0
 */

namespace ExamplePlugin;

class PostType {
  private $post_type = 'example';
  
  public function __construct() {
    add_action('init', [$this, 'register_post_type']);
  }
  
  public function register_post_type() {
    $args = [
      'public' => true,
      'label' => 'Examples',
      'supports' => ['title', 'editor', 'thumbnail'],
      'has_archive' => true,
      'rewrite' => ['slug' => 'examples'],
    ];
    
    register_post_type($this->post_type, $args);
  }
}

new PostType();
  • ✓ Always use type hints for function parameters and return types
  • ✓ Use strict comparison (===) instead of loose (==)
  • ✓ Validate and sanitize all user input
  • ✓ Use prepared statements for database queries
  • ✓ Follow WordPress coding standards
